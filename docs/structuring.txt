Structuring a FastAPI Project:
my_fastapi_project/	
├── app/	
│   ├── __init__.py
│   ├── api/
│   │   ├── router/	        #Handels API endpoints (request - response)
│   │   │   ├── __init__.py
│   │   │   └── user_router.py
│   │   │   └── item_router.py
│   │   ├── __init__.py
│   │   ├── deps.py         #Contains shared dependencies like database sessions
│   │   └── main.py         #Registers and includes API routers
│   ├── core/
│   │   ├── __init__.py	
│   │   ├── database.py     #Database connection and session setup
│   │   ├── security.py     #Security logic (hashing, JWT, auth)
│   │   └── config.py       #App configuration (env variables, settings)
│   ├── email-templates/
│   │   ├── build/          #Compiled HTML email template
│   │   │   └── mail.html
│   │   ├── src/
│   │   │   └── mail.mjml   #Source email template (MJML format)
│   ├── worker/
│   │   ├── __init__.py	
│   │   ├── celery_app.py   #Celery configuration (queue, broker)
│   │   └── tasks.py        #Background tasks (e.g., send email)
│   ├── alembic/            #Alembic config
│   │   ├── versions/
│   │   ├── env.py
│   │   ├── README
│   │   └── script.py.mako
│   ├── main.py             #The entry point of the FastAPI application
│   ├── models.py           #SQLAlchemy models
│   ├── crud.py             #Database operations (create, read, update, delete)
│   ├── schemas.py          #Pydantic schemas (data validation)
│   ├── utils.py            #Utility/helper functions
│   └── __init__.py
├── tests/
│   ├── __init__.py	
│   ├── test_main.py	
│   ├── test_users.py	
│   ├── test_items.py	
├── .env	
├── .gitignore	
├── requirements.txt	
├── README.md	
└── run.sh

client -> uvicorn  -> fastapi -> router -> dependency injection -> schema -> service -> database -> service -> router -> schema -> response -> client
